# Weather Dashboard

Приложение для отображения погоды с имитацией загрузки данных из разных источников.

## Краткое описание

Weather Dashboard — это Android-приложение, которое показывает температуру, влажность, скорость ветра и вычисляет индекс погоды. Данные загружаются параллельно с использованием корутин Kotlin, что обеспечивает быструю загрузку (~2 секунды вместо 4.5). Приложение демонстрирует работу с различными диспетчерами корутин (Main, IO, Default), обработку ошибок и автоматическое обновление данных.

## Функциональность

- Отображение температуры (15-35°C)
- Отображение влажности (40-80%)
- Отображение скорости ветра (0-20 м/с)
- Вычисление индекса погоды (тяжёлые вычисления на Dispatchers.Default)
- Параллельная загрузка данных с помощью async
- Индикатор прогресса загрузки
- Кнопка обновления данных
- Кнопка симуляции ошибки (для тестирования обработки ошибок)
- Автоматическое обновление каждые 10 секунд
- Обработка ошибок с отображением сообщения
- Сохранение состояния при повороте экрана (ViewModel)

## Технологии и библиотеки

## Контрольные вопросы

### А) В чём разница между launch и async?

**Пример кода:**
```kotlin
// launch - без возврата
viewModelScope.launch {
    saveDataToDatabase()
}

// async - с возвратом результата
val result = viewModelScope.async {
    fetchUserData()
}.await()

###Б) Что такое suspend функция?

Suspend функция — это функция, которая может приостановить своё выполнение, не блокируя поток.

**Может ли она вызываться из обычной функции?**
Нет, suspend функции можно вызывать только из других suspend функций или из корутины (launch/async).

**Почему delay() не блокирует поток?**
`delay()` приостанавливает только корутину, но не блокирует поток, позволяя другим корутинам выполняться. В отличие от `Thread.sleep()`, который блокирует поток целиком.

### В) Зачем нужны разные диспетчеры?

| Диспетчер | Для каких задач | Пример |
|-----------|-----------------|--------|
| **Dispatchers.Main** | Обновление UI | Изменение текста в TextView |
| **Dispatchers.IO** | Сеть, файлы, БД | Загрузка данных из интернета |
| **Dispatchers.Default** | Тяжёлые вычисления | Сортировка большого массива |

**Что будет, если выполнить тяжёлое вычисление на Dispatchers.Main?**
UI зависнет, приложение перестанет реагировать на нажатия, пользователь увидит сообщение "Приложение не отвечает".

### Г) Что произойдёт, если не обработать исключение в корутине?

Корутина отменится, а исключение может "всплыть" и вызвать краш приложения (FATAL EXCEPTION).

**Как корректно обрабатывать ошибки?**

```kotlin
viewModelScope.launch {
    try {
        val data = fetchData()
        updateUi(data)
    } catch (e: Exception) {
        showError(e.message)
    }
}
